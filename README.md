# Geodata Change Detection and Update Workflow Using QGIS

## Project Overview

This project demonstrates a geodata change detection and update workflow using QGIS, OpenStreetMap building data and Google Satellite imagery. Existing building footprints were visually compared against satellite imagery to identify potential discrepancies and update candidates. The project also documents FKB maintenance concepts and SOSI validation workflows used in Norwegian geodata management.

## Study Area

Nygård, Bergen, Norway

## Data Sources

- OpenStreetMap (OSM) Building Data
- Google Satellite Imagery
- QGIS

## Workflow

Satellite Imagery
↓
Existing Building Data
↓
Visual Change Detection
↓
Update Recommendation
↓
Quality Assurance Review
↓
Geodata Update

## Key Tasks

### Visual Change Detection

Compared building footprints against satellite imagery to identify potential discrepancies and update candidates.

### Update Recommendations

Created a review layer to document locations that may require further investigation and possible geodata updates.

### FKB Workflow Review

Documented how change detection results may contribute to FKB-based geodata maintenance activities.

### SOSI Workflow Review

Documented how SOSI validation supports quality assurance and integration of updated geodata.

### Change Detection Mapping

Produced a change detection map showing building footprints and potential update locations.

## Key Findings

- Potential update candidates were identified through visual comparison of building footprints and satellite imagery.
- Geodata update recommendations were documented for further review.
- Change detection workflows support geodata maintenance activities.
- FKB and SOSI concepts play important roles in Norwegian geodata management and quality assurance.
